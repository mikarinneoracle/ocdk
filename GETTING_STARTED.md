# Getting Started with OCI Infrastructure using OCDK

This guide walks you through creating OCI infrastructure using OCDK (OCI CDK) - Terraform CDK for Oracle Cloud Infrastructure.

## Prerequisites

### 1. Install Required Tools

```bash
# Node.js 18+ (check version)
node --version  # Should be 18 or higher

# Install Terraform CDK CLI globally
npm install -g cdktf-cli

# Verify installation
cdktf --version
```

### 2. Configure OCI CLI

```bash
# Configure OCI CLI (if not already done)
oci setup config

# Follow the prompts:
# - Enter your OCI user OCID
# - Enter your tenancy OCID
# - Select a region
# - Choose API key authentication
# - Enter the path to your private key (or generate a new one)
```

This creates `~/.oci/config` which OCDK will use for authentication.

## Step 1: Get Your OCI Identifiers

You'll need these OCIDs (Oracle Cloud Identifiers) for configuration:

### Get Tenancy OCID

```bash
# From OCI Console: Identity & Security > Tenancy Details
# Or via CLI:
oci iam tenancy get --tenancy-id $(oci iam region-subscription list --query 'data[0]."tenancy-id"' --raw-output)
```

### Get Compartment OCID

```bash
# List compartments
oci iam compartment list

# Get your home compartment (recommended for OCIR)
oci iam compartment list --compartment-id-in-subtree true --query 'data[?name==`YOUR_USERNAME`]' --raw-output

# Or get any compartment OCID from OCI Console:
# Identity & Security > Compartments
```

### Get Object Storage Namespace

```bash
# Get your namespace (needed for OCIR and Object Storage)
oci os ns get
```

**Note**: Namespace is usually your tenancy name, but verify with the command above.

## Step 2: Set Up Your Project

### Option A: Use Main OCDK Project

```bash
cd ~/projects/ocdk

# Install dependencies (cdktf and constructs only)
npm install

# Generate OCI Terraform provider code (creates .gen/providers/oci/)
npm run get  # This runs: cdktf get
```

**Important**: The `@cdktf/provider-oci` package is NOT installed via npm. It's generated by `cdktf get` command. The provider code will be created in `.gen/providers/oci/` directory after running `npm run get`.

### Option B: Create New Project from Test Template

```bash
cd ~/projects/ocdk/test/java-function/oci-cdk

# Install dependencies
npm install

# Download OCI Terraform provider
npm run get
```

## Step 3: Configure Environment Variables

Set the required OCI configuration:

```bash
# Required: Basic OCI Configuration
export OCI_COMPARTMENT_ID="ocid1.compartment.oc1..aaaaaaa..."
export OCI_TENANCY_ID="ocid1.tenancy.oc1..aaaaaaa..."
export OCI_REGION="eu-frankfurt-1"  # Your preferred region
export OCI_NAMESPACE="your-namespace"  # From `oci os ns get`

# Required for OCIR: Must be a non-root compartment
export OCI_OCIR_COMPARTMENT_ID="ocid1.compartment.oc1..aaaaaaa..."  # Your home compartment

# Optional: Customize resource names
export OCI_FUNCTION_APP_NAME="my-function-app"
export OCI_FUNCTION_NAME="my-function"
export OCI_OCIR_REPOSITORY_NAME="my-function-repo"

# Optional: Stack name
export OCI_STACK_NAME="my-oci-stack"
```

### Create a `.env` File (Optional)

Instead of exporting variables, you can create a `.env` file:

```bash
# Create .env file
cat > .env << EOF
OCI_COMPARTMENT_ID=ocid1.compartment.oc1..aaaaaaa...
OCI_TENANCY_ID=ocid1.tenancy.oc1..aaaaaaa...
OCI_REGION=eu-frankfurt-1
OCI_NAMESPACE=your-namespace
OCI_OCIR_COMPARTMENT_ID=ocid1.compartment.oc1..aaaaaaa...
OCI_FUNCTION_APP_NAME=my-function-app
OCI_FUNCTION_NAME=my-function
OCI_OCIR_REPOSITORY_NAME=my-function-repo
EOF

# Source it (if using a shell script to load it)
source .env
```

**Note**: `.env` files are gitignored by default. For production, use environment variables or a secrets manager.

## Step 4: Configure Terraform State Backend (Optional)

By default, Terraform state is stored locally. For production, use remote state:

### Option 1: OCI Native Backend (Recommended)

```bash
export OCI_STATE_BACKEND_TYPE="oci"
export OCI_STATE_BUCKET="tf-state"  # Bucket name (must exist)
export OCI_STATE_KEY="oci-stack/terraform.tfstate"  # State file path
```

**Prerequisites**: Create an Object Storage bucket first:
```bash
oci os bucket create \
  --compartment-id $OCI_COMPARTMENT_ID \
  --name tf-state \
  --namespace $OCI_NAMESPACE
```

### Option 2: Local State (Default)

```bash
export OCI_STATE_BACKEND_TYPE="local"
# Or simply don't set OCI_STATE_BACKEND_TYPE
```

## Step 5: Build Your Infrastructure Code

### Compile TypeScript

```bash
npm run build
```

### Preview Changes (Dry Run)

```bash
# Synthesize Terraform code (see what will be created)
npm run synth

# Preview changes (diff)
npm run diff
```

This shows you what resources will be created without actually creating them.

## Step 6: Deploy Infrastructure

### Deploy Stack

```bash
npm run deploy
```

This will:
1. Create the Terraform plan
2. Show you what will be created
3. Ask for confirmation
4. Create resources in OCI

### Verify Deployment

```bash
# List stacks
npm run list

# Check resources in OCI Console or via CLI
oci fn application list --compartment-id $OCI_COMPARTMENT_ID
oci artifacts container repository list --compartment-id $OCI_OCIR_COMPARTMENT_ID
```

## Step 7: Add Resources to Your Stack

Edit `lib/oci-stack.ts` to add more resources. After running `npm run get`, you'll have TypeScript types for all OCI resources.

### Example: Add an Object Storage Bucket

```typescript
import { ObjectstorageBucket } from '@cdktf/provider-oci/lib/objectstorage-bucket';

// Inside your stack constructor:
const bucket = new ObjectstorageBucket(this, 'MyBucket', {
  compartmentId: config.compartmentId,
  namespace: config.namespace,
  name: 'my-bucket',
  accessType: 'NoPublicAccess',
});
```

### Example: Add OCI Function Application

```typescript
import { FunctionsApplication } from '@cdktf/provider-oci/lib/functions-application';

const functionApp = new FunctionsApplication(this, 'FunctionApp', {
  compartmentId: config.compartmentId,
  displayName: config.functionAppName,
});
```

**Note**: After adding new resources, run `npm run get` again if you need new provider types.

## Step 8: Update Infrastructure

```bash
# Make changes to lib/oci-stack.ts
npm run build
npm run diff  # Preview changes
npm run deploy  # Apply changes
```

## Step 9: Destroy Infrastructure (When Done)

```bash
npm run destroy
```

**Warning**: This will delete all resources created by the stack!

## Troubleshooting

### Error: "OCIR repository cannot be created in root compartment"

**Solution**: Set `OCI_OCIR_COMPARTMENT_ID` to a non-root compartment (your home compartment).

```bash
export OCI_OCIR_COMPARTMENT_ID="ocid1.compartment.oc1..aaaaaaa..."
```

### Error: "Provider not found" or TypeScript errors

**Solution**: Run `npm run get` to generate provider code:

```bash
npm run get  # Generates .gen/providers/oci/
npm run build
```

**Note**: The `@cdktf/provider-oci` package is NOT installed via npm. It's generated by `cdktf get`. If you see errors about missing `@cdktf/provider-oci`, make sure you've run `npm run get` first.

### Error: "@cdktf/provider-oci not found" during npm install

**Solution**: Remove `@cdktf/provider-oci` from `package.json` devDependencies. It's not an npm package - it's generated by `cdktf get`:

```bash
# The provider code is generated, not installed
npm run get  # This creates .gen/providers/oci/
```

### Error: Authentication failed

**Solution**: Verify OCI CLI is configured:

```bash
oci iam region-subscription list
```

If this fails, reconfigure:
```bash
oci setup config
```

### Error: Permission denied

**Solution**: Ensure your OCI user has the necessary IAM policies:

- `manage all-resources` (for full access)
- Or specific policies for Functions, Object Storage, API Gateway, etc.

## Quick Reference

CDK-style commands (run from the **OCDK project root**):

| Command | Description |
|---------|-------------|
| `npm run ocdk -- deploy` or `npm run deploy` | Deploy infrastructure |
| `npm run ocdk -- diff` or `npm run diff` | Preview changes |
| `npm run ocdk -- synth` or `npm run synth` | Generate Terraform code |
| `npm run ocdk -- destroy` or `npm run destroy` | Destroy infrastructure |
| `npm run ocdk -- list` or `npm run list` | List stacks |
| `npm run ocdk -- get` or `npm run get` | Download Terraform providers |

**Note:** `npx ocdk` does not work (no published package). Use `npm run ocdk -- <command>` or `npm run <command>` from the project root. For the test app in `test/java-function/oci-cdk/`, run `npm run deploy` from that directory.

Other:
| `npm install` | Install dependencies (includes cdktf-cli) |
| `npm run build` | Compile TypeScript |
